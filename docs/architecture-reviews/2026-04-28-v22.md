# Architecture review — 2026-04-28 (post-v0.22)

**Cycle:** first v2 re-evaluation cycle.
**Cadence trigger:** end of v0.22 (auth-backend split + Thalyn identity
+ first-run wizard + day-divider digest + capability-delta auth row).
**Previous review:** [`2026-04-26-v15.md`](./2026-04-26-v15.md) — the
last v1 cycle, before the v2 baseline rewrite.

This cycle covers the v2 baseline phases v0.20 (data-model + agent
registry foundation), v0.21 (eternal-thread durability + rolling
summarizer), and v0.22 (auth-backend split + Thalyn identity).

## Method

Inventoried each accepted ADR. For every load-bearing dependency, web-
searched current upstream state, compared to what shipped or pinned in
v0.20–v0.22, classified as **still-current**, **refine**, or
**supersede**. Sources cited inline.

---

## Findings

### ADR-0001 — Tauri 2 — **still-current**

Tauri 2.10.3 is the production line (March 2026); v2.11 in flight, no
cuts since the v0.15 review. The v2 IPC rewrite (raw payloads, custom
protocols, `tauri::ipc::Channel`) is settled — the v0.20–v0.22 commits
added five new commands (`auth_list / auth_probe / auth_set /
thread_recent / digest_latest`) on top of the existing
`BrainSupervisor` plumbing without surface changes. The Windows IPC
perf risk from v0.6 / v0.12 / v0.15 reviews remains untested; carries
into v0.23+. ([Tauri release feed](https://v2.tauri.app/release/),
[GitHub releases](https://github.com/tauri-apps/tauri/releases))

No action.

### ADR-0002 — React 19 + shadcn/ui + Tailwind v4 — **refine**

The dep-resolution layer is unchanged at major-version level, but the
**shadcn/ui line moved meaningfully** between the v0.15 review and
v0.22:

- **`forwardRef` removed.** Every shadcn primitive now uses native
  React-19 `ref` props, with `data-slot` attributes for styling.
  Migration is opt-in via the `remove-forward-ref` codemod; v3 + React
  18 still works.
- **HSL → OKLCH** colour tokens (we already moved per ADR-0013, so
  upstream caught up).
- **Default style deprecated → `new-york`**.
- **`toast` deprecated → `sonner`**.

Our v0.20–v0.22 components (first-run wizard, thread-digest greeting,
capability-delta auth row) were authored without `forwardRef` so the
upstream forward-ref purge doesn't bite us. The `sonner` migration is
v0.23+ work whenever a toast is needed; we don't ship one yet. The
`data-slot` audit lands when we touch each primitive next.

No new ADR; ADR-0013 picks up an addendum noting the forward-ref-free
authoring convention plus the `sonner` pointer for whenever toasts
arrive. ([shadcn/ui Tailwind v4](https://ui.shadcn.com/docs/tailwind-v4),
[shadcn/ui changelog](https://ui.shadcn.com/docs/changelog))

### ADR-0009 — Memory: Mem0 + LangGraph + project files — **refine (defer Mem0 longer)**

Mem0 Python 1.0.4.6 (Mar 15, 2026) is active and shipping; +26% over
OpenAI Memory on LOCOMO. **But CVE-2026-0994 is open** — Mem0's
`protobuf` pin (`>=5.29.0,<6.0.0`) blocks the patched version, so
adopting Mem0 today would import a known CVE into the brain. The
ADR-0022 `FTS5-now / Mem0-later` deferral was the right call; this
cycle's finding is "the deferral is more right now than it was in
v0.21."

ADR-0009 carries an addendum noting the CVE; Mem0 stays on the
roadmap behind the same `thread.search` / memory contract. Re-check
at the v0.25 cycle.
([Mem0 changelog](https://docs.mem0.ai/changelog),
[CVE-2026-0994 issue](https://github.com/mem0ai/mem0/issues/3963))

### ADR-0020 — brain auth-backend split — **refine**

Shipped this cycle as **Proposed**. The spike doc
(`docs/spikes/2026-04-28-claude-cli-auth.md`) confirmed the SDK
inherits the bundled CLI's auth state and the `claude auth status
--json` shape is stable.

The newly-discovered wrinkle: **Anthropic enforced a policy in early
April 2026 that the Agent SDK requires API-key auth and explicitly
prohibits Pro/Max subscription billing**. The bundled CLI itself
still authenticates fine against subscription auth (it's the user's
own client of the Claude product, not the SDK), but the SDK's *API*
calls are routed through the user's API account, not their Pro/Max
subscription. Issue #559 tracks Max-plan billing for the SDK.

The implication for ADR-0020:
- Our subscription-default works at the *CLI* layer — `claude
  auth status` reports `loggedIn: true` and the bundled binary
  delegates calls to whatever Claude product the user is signed into.
- Whether Claude API costs hit Pro/Max billing or burn API credit
  depends on Anthropic's accounting, which is outside Thalyn's
  control. The user's choice between subscription and API key is
  about *credential source*, not about *billing target*.
- The ADR-0020 wording should add a paragraph noting this. The
  user-visible UX is unchanged (the wizard's Claude-subscription
  option still works); the subtlety belongs in docs and the
  capability-delta dialog's explanatory copy.

ADR-0020 stays **Proposed** with an addendum; no supersession.
([SDK overview](https://platform.claude.com/docs/en/agent-sdk/overview),
[Issue #559](https://github.com/anthropics/claude-agent-sdk-python/issues/559))

### Worker model routing (ADR-0023, to be drafted in v0.24) — **clarify defaults**

The user-memory `project_worker_routing` says "v1 default = all
workers on Opus 4.7". With Opus 4.7 (Apr 16, 2026), Sonnet 4.6 (Feb
17, 2026), and Haiku 4.5 (Jan 2026) as the current Anthropic line,
the v0.24 routing ADR should pin:
- `coding`, `research`, `default` → Opus 4.7
- `writing`, `quick` → Sonnet 4.6 (price/perf tier)
- Per-project override → Haiku 4.5 if the user wants the budget tier

**Operational note:** Sonnet 4 + Opus 4 retired Apr 20, 2026 (notice
Apr 14). Any pinned-model code paths referencing those names will
break. We don't pin them today (`AnthropicProvider.DEFAULT_MODEL` is
`claude-sonnet-4-6`), so we're clean — but the routing-ADR work in
v0.24 should add a CI gate that fails if a retired model id appears
in source.
([Models overview](https://platform.claude.com/docs/en/about-claude/models/overview),
[Model deprecations](https://platform.claude.com/docs/en/about-claude/model-deprecations))

   that clarifies the credential / billing distinction.

   CVE-2026-0994; revisit at v0.25.
3. **ADR-0013 addendum** — forward-ref-free + `data-slot` authoring
   convention; `sonner` pointer for future toasts.
4. **ADR-0017 addendum** — OTel GenAI semconv experimental status;
   dual-emission once it stabilises.
5. **v0.24 routing ADR** — when drafted, lock in Opus 4.7 / Sonnet
   4.6 / Haiku 4.5 tiers; CI gate on retired model ids (Sonnet 4 /
   Opus 4 reject).
6. **`02-architecture.md` topology check** — no shifts this cycle;
   the auth-backend trait split is captured in §7 and was the v0.22
   load-bearing change.

These five addenda are non-blocking; the existing ADRs stay accurate
without them, but capturing them now means the next cycle starts from
a more current baseline.
