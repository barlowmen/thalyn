# Architecture review — 2026-04-26

**Cycle:** first re-evaluation cycle (post-v0.3, per `` §6).
**Cadence trigger:** end of v0.3.

This review re-validates the active stack against current upstream
state. Deviations large enough to change a decision get filed as new
ADRs that supersede the old; smaller refinements are noted in-place
on the existing ADR.

## Method

For each load-bearing dependency: pull its current release notes /
upstream state, compare to what we shipped or pinned in v0.1–v0.3,
classify as **still current**, **refine**, or **supersede**. Sources
are cited inline.

---

## Findings

### ADR-0001 — Tauri 2 — **still current**

Tauri 2 stable shipped 2024-10-08; the project explicitly states
**no more breaking changes are expected** post-stable. Our pin
(`tauri = "2"` resolving to 2.10.x) is on the current line. The
capabilities + sidecar surface we used in v0.1 (`shell:allow-execute`,
`externalBin`) is still the canonical 2026 shape.
([release notes](https://v2.tauri.app/release/),
[stable announcement](https://v2.tauri.app/blog/tauri-20/))

No action.

### ADR-0002 — React 18 + shadcn/ui + Tailwind + Vite — **refine (already)**

We shipped React 19 + Tailwind v4 + shadcn/ui for Tailwind v4 in
v0.2, ahead of ADR-0002's "v0.6 review" note. The combination has
been smooth in v0.3 (paste-API-key flow, command palette, dialog,
input, label, badge primitives all built without surprises). The
ADR-0002 "Negative" note (Tailwind v4 / React 19 upgrade due v0.6) is
already retired by the v0.2 work. No supersession needed; the
*Refinement at v0.2 implementation* note is the correct place to
capture that.

Action: add a brief "Refinement at v0.2/v0.3 implementation" note to
ADR-0002 documenting that we adopted React 19 + Tailwind v4 ahead of
schedule (see follow-up commit).

### ADR-0007 — Orchestration: LangGraph 1.0 + Claude Agent SDK — **on schedule**

LangGraph orchestration lands in v0.4; we have not yet wired it.
The Claude Agent SDK is moving fast: latest stable is **0.1.68**
released **2026-04-25** (one day before this review).
Notable v0.1.x deltas since our `>=0.1.4` pin:

- **Claude Code CLI is now bundled with the SDK package** — no
  separate `npm i -g @anthropic-ai/claude-code` step. This removes
  a packaging concern for the eventual PyOxidizer/Briefcase pass.
- **OpenTelemetry trace context propagation** — useful when ADR-0017
  observability lands (v0.14).
- **`skills` parameter on `ClaudeAgentOptions`** — relevant once we
  expose user-authored skills (per F9.2).
- **SessionStore parity with TS SDK** — useful when multi-turn
  history threading lands.
([SDK on PyPI](https://pypi.org/project/claude-agent-sdk/),
[GitHub releases](https://github.com/anthropics/claude-agent-sdk-python/releases))

Action: bump `brain/pyproject.toml` pin to `claude-agent-sdk>=0.1.68,<0.2`
in a follow-up commit. No ADR change.

### ADR-0012 — Provider abstraction — **refined in v0.3**

The implementation deviated from the Rust-trait sketch (narrower
trait; Python uses single `stream_chat`; `ToolCallNormalizer`
deferred). The 2026-04-26 commit `93d2929` added a *Refinement at
v0.3 implementation* note to ADR-0012 capturing all three deltas;
the **Decision** is unchanged. No supersession.

### ADR-0005 — IPC: NDJSON + JSON-RPC 2.0 — **still current**

The streaming-notification refactor in v0.3 (request handlers can
emit notifications mid-flight; the supervisor routes them through a
callback) fits the original ADR cleanly. Wire shape unchanged; only
the handler surface in the Python dispatcher grew a streaming variant
and the Rust supervisor grew a `call_streaming` shape. ADR remains
authoritative.

### ADR-0006 — PyOxidizer packaging — **deferred (still on horizon)**

Not exercised in v0.3; we still spawn the brain via `uv run python -m
thalyn_brain`. The CLI-bundled-with-SDK change above lowers the
PyOxidizer risk slightly (fewer external dependencies to chase). No
action this cycle; revisit when v0.x reaches the packaging phase.

### ADR-0013 — Design system (OKLCH + Geist + Motion) — **still current**

OKLCH tokens and the @theme inline bridge survived first contact
with shadcn/ui's component generators; Tailwind v4's first-class
`@theme inline` block is a much cleaner home for the bridge than
v3's `tailwind.config.js` ever was. Reduced-motion / reduced-
transparency tokens fired correctly during the visual-regression
runs. No drift.

### ADR-0014, 0015, 0016 — Documentation, commit hygiene, license — **still current**

Leakage scanner caught two real near-misses during v0.3 (one
agent-attribution trailer reference in narrative, one bare
build-plan-slug slip) and one false positive (the on-disk filename
of the build plan, already exempted in the v0.1 narrowing pass).
MIT is unchanged; the going-public re-evaluation lands later.

### ADR-0017 — Observability — **on schedule**

OpenTelemetry GenAI instrumentation lands in v0.14. The Claude Agent
SDK now ships W3C trace-context propagation for the CLI subprocess
(see SDK note above), which dovetails neatly — when we wire OTel into
the brain's request handlers, the spans will already carry through to
the Anthropic-side traffic without additional plumbing.

### Other ADRs (0003, 0004, 0008–0011) — **untouched**

Editor (Monaco), brain process model (Python sidecar), durable
execution (SqliteSaver), memory (Mem0), browser (sidecar Chromium),
sandbox tiers — none have been exercised yet in v0.1–v0.3. Each
remains valid as a forward-looking decision.

## Recommended actions

1. **Add a refinement note to ADR-0002** documenting the early Tailwind v4 + React 19 adoption (low-effort, in this cycle).
2. **Bump `claude-agent-sdk` pin** to `>=0.1.68,<0.2` (follow-up commit).
3. **Plan**: thread `skills` and `SessionStore` into the brain when v0.6 (sub-agents) and v0.11 (memory) land — both upstream surfaces are now stable enough to consume.
4. **No ADR supersessions** this cycle.
