## Findings (delta only)

### ADR-0007 — Orchestration: LangGraph 1.0 + Claude Agent SDK — **still-current**

The v0.25 review tagged LangGraph **1.1.10 (2026-04-27)**. Upstream
shipped **`langgraph==1.2.0a1` (2026-04-29)** roughly an hour before
this review window opens. Headline changes are additive: graceful
shutdown/drain support, native v2 projections (`custom`, `updates`,
`checkpoints`, `debug`, `tasks`), `DeltaChannel` blob sentinels, the
streaming-transformer infrastructure, decoupling
`run.output/interrupted/interrupts` from `ValuesTransformer`, plus
`checkpoint==4.1.0a1` alongside.

Per LangChain's stated release policy the 1.x line is additive — pinning
to stable means the alpha doesn't reach our dep graph until GA. The
brain stays on 1.1.x; ADR-0007 stays current. Worth a focused look once
1.2 stabilises (the v2 projections are something the worker drawer's
plan-update stream could feed off cleanly), but no posture shift now.
([LangGraph releases](https://github.com/langchain-ai/langgraph/releases),
[release policy](https://docs.langchain.com/oss/python/release-policy))

No action.

### ADR-0007 (continued) — Claude Agent SDK — **still-current with one re-pin note**

Eight patch releases in eleven days; latest is **`claude-agent-sdk-python
v0.1.71` (2026-04-29)**. None breaking. The window the v0.25 review
covered ended around v0.1.63; the deltas since:

- **v0.1.71** (Apr 29) — `allowedDomains` / `deniedDomains` /
  `allowManagedDomainsOnly` / `allowMachLookup` fields on
  `SandboxNetworkConfig` reach Python parity with the TS schema.
  Touches ADR-0011 (sandbox tiers) the day we plumb network policy
  through to a worker; today nothing reads them.
- **v0.1.70** (Apr 28) — floor bumped to `mcp>=1.19.0`. Older MCP
  versions silently dropped `CallToolResult` from in-process tool
  handlers, which would manifest as silently-missing tool results in
  the worker drawer's action log. When we next bump the SDK pin we
  should verify the resolved `mcp` version satisfies the floor.
- **v0.1.67** (Apr 25) — trio compatibility regression fix. We're on
  asyncio; not applicable.
- **v0.1.65** (Apr 23) — batch session summaries
  (`SessionStore.list_session_summaries`), `import_session_to_store`,
  `display` field on `ThinkingConfig` (lets us un-omit Opus 4.7 thinking
  summaries when we eventually surface them), and two new content-block
  types: `ServerToolUseBlock` and `AdvisorToolResultBlock`. The new
  blocks were *silently dropped* by every prior version, which means our
  current message-handling layer may already be losing them. Worth a
  one-line audit when we next touch the chat-chunk pipeline.
- **v0.1.64** (Apr 20) — SessionStore protocol at TS parity, conformance
  test harness, S3 / Redis / Postgres reference adapters in `examples/`.

ADR-0020 (auth-backend split) sees no change — no release notes touch
the subscription-auth posture. The user's Claude subscription remains
the v1 default brain auth.

No forced shift; the new content-block audit and the `mcp>=1.19.0`
floor go on the next-SDK-bump checklist.
([Claude Agent SDK releases](https://github.com/anthropics/claude-agent-sdk-python/releases))

### ADR-0019 — Browser engine v2 (`tauri-apps/cef-rs`) — **still-Proposed; bundling story unchanged**

Posture check before v0.29 lights up the swap. The crate is *actively*
maintained — three CEF version bumps in nine days (146.0.11 → 146.0.12
→ 147.0.9 → 147.0.10), latest **`cef-v147.1.0+147.0.10` (2026-04-26)**.
Release-plz on a near-daily cadence; abandonment risk is low.

The bundling blocker the v0.25 review flagged is unmoved. Tracking
issues `#209` (bundling tracker) and `#192` (Tauri integration tracker)
both stay where they were — `#209` last had Nov 2025 discussion of
`cargo binstall` + release artefacts of `export-cef-dir` output as a
CDN-free path, but no commits since the v0.25 review address downstream
bundling. The v0.29 phase will land on `export-cef-dir` + CDN download,
the same posture ADR-0019 already records.

`#364` (`Invalid UTF-16 string` stderr spam, 49+ messages per launch)
is pre-existing and would surface noisily in our logs. Cosmetic; noted
for v0.29's logging filter.

ADR-0019 stays *Proposed*. Re-check at v0.33 (the next cycle that
intersects the bundling story; the architecture-review attaches to
the Voice Input phase, not the v0.32 spike that feeds it).
([cef-rs releases](https://github.com/tauri-apps/cef-rs/releases),
[#209](https://github.com/tauri-apps/cef-rs/issues/209),
[#192](https://github.com/tauri-apps/cef-rs/issues/192),
[#364](https://github.com/tauri-apps/cef-rs/issues/364))

No action.
