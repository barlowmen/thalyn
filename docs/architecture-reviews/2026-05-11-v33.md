# Architecture review — 2026-05-11 (post-v0.33)

**Cycle:** fourth v2 re-evaluation cycle.
**Cadence trigger:** end of v0.33 (Voice Input).
**Previous review:** [`2026-04-29-v28.md`](./2026-04-29-v28.md) — third v2
cycle, post-v0.28 (worker / lead drawers + direct lead chat).

This cycle covers v0.29 (in-process CEF foundation), v0.30 (CEF engine
swap in-process), v0.31 (multi-project juggling + morning digest UI),
v0.32 (voice integration spike), and v0.33 (voice input — local
Whisper engine, model store + bundle preload, cpal mic capture +
streaming hot path, level meter, project vocabulary, continuous-listen
mode with simple-VAD, Deepgram + MLX skeletons, mic-permission deep-
link, M4 bake-off re-run with delegation fixture, ADR-0025 promoted).
Because the prior review landed only 12 days ago the search horizon
is narrow; the voice stack is in scope as load-bearing for this cycle.

## Method

Re-confirmed each ADR's posture against upstream state. Voice stack
(`whisper-cpp-plus`, `whisper.cpp`, MLX-Whisper, faster-whisper,
Piper, Deepgram) gets a full pass; LangGraph / Claude Agent SDK /
cef-rs get a delta-pass against the v0.28 review's window. Other
load-bearing deps re-checked at the smoke level. Sources cited
inline.

### ADR-0007 — LangGraph + Claude Agent SDK — **still-current; one deprecation to grep**

**LangGraph.** Stable still on 1.1.x; 1.2 line is at **1.2.0a7
(2026-05-04)** — five alpha bumps in the window, none GA. v3
streaming API with typed per-channel projections, DeltaChannel
reduced-overhead checkpointing, per-node timeouts + idle, node-
level error handlers, graceful shutdown. Hold on 1.1.x; revisit
when 1.2 GA lands. The v3 streaming protocol is a tracking note
for whenever we next touch a streaming surface.
([releases](https://github.com/langchain-ai/langgraph/releases))

**Claude Agent SDK.** Ten releases in the window — **v0.1.71 →
v0.1.81 (2026-05-11, today)**. Headline changes:

- **v0.1.77 (May 8) — deprecation, action required.** `"Skill"` in
  `allowed_tools` is deprecated; use the `skills` option in
  `ClaudeAgentOptions` instead. A grep across Thalyn's brain for
  `"Skill"` literal in `allowed_tools` will catch any call sites
  that need to migrate before a future SDK pin bump.
- **v0.1.74 (May 6) — additions worth noting.** Hook event
  streaming (`include_hook_events`), deferred-hook-decision
  support, strict MCP config option, `xhigh` effort level for
  Opus 4.7, subprocess cleanup on exit.
- **v0.1.73 (May 4) — additions worth noting.**
  `session_store_flush: "batched" | "eager"` — eager flush is
  directly useful for real-time transcript delivery (the voice
  pipeline's natural shape).
- **v0.1.76 (May 6) — small fixes.** `api_error_status` on
  `ResultMessage`; `ToolPermissionContext.suggestions`
  deserialization fix.

Non-breaking otherwise. ADR-0007 stays current; the next SDK pin
bump needs:
1. The `"Skill"` → `skills` migration grep.
2. The same `mcp>=1.19.0` floor + `ServerToolUseBlock` /
   `AdvisorToolResultBlock` audit the v0.28 review already filed.
3. Optional: consider adopting `xhigh` effort and eager session
   flush — both fit Thalyn's surfaces cleanly.
([releases](https://github.com/anthropics/claude-agent-sdk-python/releases))

### ADR-0009 — Memory: Mem0 + LangGraph checkpoints — **new caveat**

Mem0 is **not** integrated into Thalyn today (per the v0.27+ scope
note in PROGRESS.md, Mem0 lights up behind the existing
`memory.search` surface if and when we decide to). New signal from
this cycle's research:

- **Mem0 v2.0.2 (2026-05-07)** is the current Python release; v2
  line is recent.
- **CVE-2026-0994** (issue #3963) — Mem0 pins `protobuf>=5.29.0,<6.0.0`,
  blocking transitive upgrade to patched protobuf 6.x. If Thalyn
  ever lights Mem0 up, the protobuf transitive needs an explicit
  audit.

ADR-0009 stays current (we explicitly don't depend on Mem0 yet);
the CVE is filed as a "before we light Mem0 up" prerequisite.
([Mem0 issue](https://github.com/mem0ai/mem0/issues/3963))

### ADR-0018 — Python sidecar packaging (PyInstaller) — **breaking-ish patch**

**PyInstaller v6.20.0 (2026-04-22)** has two contract changes
worth flagging:

- **Bootloader env-var contract changed.** `_PYI_ARCHIVE_FILE` is
  the new inherited-environment detection signal; if a bundle
  ever self-restarts, `PYINSTALLER_RESET_ENVIRONMENT` must be set
  before spawning the restart. Thalyn's brain sidecar doesn't
  self-restart today, but worth a tagged note.
- **Multiprocessing run-time hook cleanup.** Custom `Popen`
  overrides removed; this is the bigger watch-item for any code
  path that spawns child processes through `multiprocessing`.

Action: **bundle smoke** before the next phase that touches
packaging (likely v0.34 if conversational onboarding adds any new
brain-side code paths). ADR-0018 stays current; no rewrite
needed.
([PyInstaller releases](https://github.com/pyinstaller/pyinstaller/releases))

## Net delta from v0.28

Five phases, zero forced posture shifts. The big v2 architectural
moves of this window (in-process CEF embedding in v0.30, multi-
project juggling in v0.31, the local-Whisper voice pipeline + its
UX surface in v0.33) all landed within the shape ADR-0019,
ADR-0021, ADR-0026, and ADR-0025 specified. The most consequential
single artefact of the cycle is the M4-measured latency caveat now
recorded in ADR-0025: the headline ≤ 250 ms Apple-Silicon budget
requires Core ML or MLX, not just Metal — a clarification the v0.32
spike's interpolated numbers couldn't surface.

The field is still moving in additive directions. Voice stack picks
are stable. The one note worth carrying forward: `whisper-cpp-plus`
has low maintenance velocity, and the Core ML gap is ours to close
either way. The decision between upstream-PR and vendor-fork is
ready to make whenever the Apple-Silicon-Core-ML row on the going-
public list comes due.
